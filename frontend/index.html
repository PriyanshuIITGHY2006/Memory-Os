Scalable Persistent Memory for LLMs
How MemoryOS maintains coherence across thousands of turns without degrading token efficiency or recall accuracy.

The Stationarity Problem

Naive Approach — O(N) token cost
Concatenating full conversation history grows proportionally with turn count. At 4000 turns this saturates the context window and explodes API cost.
MemoryOS — O(1) token cost
A relevance-scored budget system selects top-K memory nodes by score and fills a fixed character budget. Token cost per turn is constant regardless of length.

I. Adaptive Relevance Scoring with Surprise-Weighted Forgetting

Extended Memory Relevance Score
$$\text{score}(m_i \mid Q, t) = \alpha \cdot \underbrace{\frac{|Q \cap W_i|}{|Q|}}_{\text{keyword}} + \beta \cdot \underbrace{e^{-\lambda_{\text{eff}}(m_i)\,\Delta t}}_{\text{adaptive recency}} + \gamma \cdot \underbrace{\frac{\ln(1+f_i)}{\ln(101)}}_{\text{frequency}} + \delta \cdot \underbrace{\sigma_i}_{\text{salience}}$$
Adaptive Decay Rate: $\;\lambda_{\text{eff}}(m_i) = \lambda_{\text{base}} \cdot \bigl(1 - \alpha_s \cdot \sigma_i\bigr)$

$\sigma_i = \min\!\bigl(d_{\cos}(m_i,\, \text{NN}(m_i)),\; 1\bigr)$  —  cosine distance to nearest neighbour at write time (information-theoretic surprise)

$\alpha_s = 0.70$  →  $\sigma_i = 1 \Rightarrow \lambda_{\text{eff}} = 0.30\lambda$  (half-life 3.3× longer for novel memories)
$\alpha = 0.45,\; \beta = 0.28,\; \gamma = 0.18,\; \delta = 0.09$

II. Personalized PageRank Spreading Activation

Graph-Topology-Aware Retrieval Re-ranking
$$r_v^{(0)} = \text{score}(m_v \mid Q, t) \cdot \mathbf{1}[v \in K]$$ $$r_v^{(k+1)} = (1-d)\,r_v^{(0)} \;+\; d \sum_{u \in \mathcal{N}(v)} \frac{w_{uv}}{\sum_j w_{uj}} \cdot r_u^{(k)}$$ $$\text{final\_score}(v) = 0.6\,r_v^{(K)} + 0.4\,r_v^{(0)}$$
$K$ = retrieved seed set  ·  $d = 0.25$ (damping)  ·  $k = 4$ iterations
Entities reachable from multiple high-score seeds gain activation even if not directly retrieved. Surfaces implicit associations the query didn't target directly.

III. Hebbian Co-Activation Edge Learning

Hebb's Rule on the Knowledge Graph
$$w_{ij}^{(t+1)} = w_{ij}^{(t)} + \eta \cdot a_i^{(t)} \cdot a_j^{(t)}$$ $$w_{ij}^{(t+1)} = w_{ij}^{(t)} \cdot (1 - \delta_H) \quad \text{(decay on unused edges)}$$
$\eta = 0.10$ (learning rate)  ·  $\delta_H = 0.005$ (per-turn decay)
Applied to all entity pairs co-retrieved in the same turn. After $\theta = 3$ co-activations, a permanent RELATED_TO edge is written to Neo4j. The graph learns which associations are genuinely meaningful vs coincidental.

IV. Complementary Learning Systems Consolidation

CLS Criterion (McClelland et al., 1995)
$$m_i \in \mathcal{C}_t \iff \sigma_i \geq \theta_\sigma \;\wedge\; (t - t_{\text{origin}}(m_i)) \geq \theta_t$$
$\theta_\sigma = 0.65$  ·  $\theta_t = 10$ turns

Hippocampal buffer (ChromaDB)  →  fast binding, verbatim, high-capacity, fades fast
Neocortical graph (Neo4j)  →  slow, compressed, semantic, persistent

Every $N=20$ turns, a background LLM pass extracts entities, events, and knowledge from high-surprise aged episodes and writes them to the graph. Source episodes are marked consolidated=True.

V. Greedy Context Selection

Fixed-Budget Selection
$$S^* = \arg\max_{S \subseteq \mathcal{M}} \sum_{m_i \in S} \text{score}(m_i \mid Q,t) \quad \text{s.t.} \quad \sum_{m_i \in S} |m_i| \leq B$$
Solved greedily by sorting $\mathcal{M}$ descending by score and filling budget $B$. Independent of $|\mathcal{M}|$.
Complexity
$$\text{tokens}_{\text{naive}}(N) = \Theta(N) \qquad \text{tokens}_{\text{MemoryOS}}(N) = \Theta(1)$$

VI. Entity Resolution

Duplicate Confidence
$$c(e_a, e_b) = \mathbf{1}[\text{rel}_a = \text{rel}_b] \cdot 0.40 + \mathbf{1}[e_a \subset e_b] \cdot 0.50 + \text{lev}(e_a, e_b) \cdot 0.40 + \mathbf{1}[\text{possessive}(e_a)] \cdot 0.60$$
Pairs with $c \geq 0.40$ are flagged for merge.

VII. Request Processing Pipeline

1
Turn Increment + Session Tracking
Atomic Neo4j counter write. SessionSummaryManager records activity for idle-based session summarisation.
2
Surprise-Weighted Proactive Retrieval
ChromaDB cosine search over last 3 buffer turns. Surprise $\sigma_i$ stored at write time gates adaptive decay. High-surprise memories surface despite age.
3
PPR-Enhanced Context Assembly
ContextBuilder scores nodes with extended formula, then runs 4-iteration Personalized PageRank to surface implicit graph neighbours. Greedy fill to fixed budget ~350 tokens.
4
LLM Inference — Groq API
9-tool schema + session context injected. Agentic loop up to 4 rounds. llama-3.3-70b-versatile. Phase 2 streams final response token by token via SSE.
5
Tool Dispatch + Contradiction Detection
Graph writes to Neo4j. Background thread checks new profile values against stored ones — flags conflicts as Contradiction nodes.
6
Hebbian Strengthening + CLS Tick
Co-retrieved entities get edge weight increment $+\eta$. CLSConsolidationEngine checks if $\geq 20$ turns since last consolidation and triggers background episode→graph promotion.

Storage Architecture

Neo4j AuraDB — Structured Memory
Graph database storing the User World Model as a labeled property graph. Node labels: User, Entity, Event, Knowledge, Preference. Enables multi-hop traversal and structural deduplication via Cypher merge.
ChromaDB — Episodic Memory
Local persistent vector store for full conversation episodes. Cosine similarity search returns semantically relevant past exchanges. Complements Neo4j with unstructured long-term recall.
